ans.) Traditionally, JavaScript was an interpreted language. However, modern JavaScript engines (like V8 in Chrome) use Just-In-Time (JIT) compilation, where the code is compiled into machine code at runtime for better performance.

ans.) The `do...while` loop is similar to the `while` loop, but it guarantees that the loop body will execute at least once, because the condition is checked *after* the first iteration.
eg.) `let i = 0;`
    `do {`
    `  console.log(i);`
    `  i++;`
    `} while (i < 0);` // Prints 0, then condition is false, loop stops.

81.) What is the `break` statement?
ans.) The `break` statement is used to terminate a loop (or `switch` statement) immediately. Execution continues with the statement immediately following the terminated loop.
eg.) `for (let i = 0; i < 10; i++) {`
    `  if (i === 5) {`
    `    break; // Exit the loop when i is 5`
    `  }`
    `  console.log(i);`
    `}`
    `// Output: 0, 1, 2, 3, 4`

82.) What is the `continue` statement?
ans.) The `continue` statement is used to skip the current iteration of a loop and move to the next iteration.
eg.) `for (let i = 0; i < 5; i++) {`
    `  if (i === 2) {`
    `    continue; // Skip when i is 2`
    `  }`
    `  console.log(i);`
    `}`
    `// Output: 0, 1, 3, 4`

83.) What is the `switch` statement?
ans.) The `switch` statement is used to perform different actions based on different conditions. It evaluates an expression and compares the result to `case` labels.
eg.) `let day = "Monday";`
    `switch (day) {`
    `  case "Monday":`
    `    console.log("Start of week.");`
    `    break;`
    `  case "Friday":`
    `    console.log("End of week.");`
    `    break;`
    `  default:`
    `    console.log("Mid-week.");`
    `}`

84.) What is the purpose of `default` case in a `switch` statement?
ans.) The `default` case is executed if none of the `case` values match the expression. It's optional.

85.) What is `document.write()`?
ans.) `document.write()` is a method used to write content directly to an HTML document. It's generally not recommended for modern web development, especially after the page has loaded, as it can overwrite the entire document content.
eg.) `document.write("<h1>Hello from JavaScript!</h1>");`

86.) What is `innerHTML`?
ans.) `innerHTML` is a property of DOM elements that allows you to get or set the HTML content (including tags and text) inside that element. It's commonly used to dynamically update parts of a web page.
eg.) `document.getElementById('myDiv').innerHTML = '<h2>New Content</h2>';`

87.) What is `textContent`?
ans.) `textContent` is a property of DOM elements that allows you to get or set just the text content of an element, ignoring any HTML tags. It's safer than `innerHTML` if you only want to insert plain text, as it prevents XSS attacks.
eg.) `document.getElementById('myParagraph').textContent = 'This is plain text.';`

88.) How do you add a CSS class to an element using JavaScript?
ans.) You can use the `classList` property and its `add()` method.
eg.) `document.getElementById('myElement').classList.add('active');`

89.) How do you remove a CSS class from an element using JavaScript?
ans.) You can use the `classList` property and its `remove()` method.
eg.) `document.getElementById('myElement').classList.remove('inactive');`

90.) How do you toggle a CSS class on an element using JavaScript?
ans.) You can use the `classList` property and its `toggle()` method. If the class exists, it removes it; otherwise, it adds it.
eg.) `document.getElementById('myElement').classList.toggle('highlight');`

91.) What is the `window` object?
ans.) The `window` object represents the browser's window. It is the global object for client-side JavaScript. All global variables, functions, and objects (like `document`, `location`, `history`, `setTimeout`) are members of the `window` object.

92.) What is the `document` object?
ans.) The `document` object represents the HTML document loaded in that window. It is the entry point to the web page's content (the DOM tree). Through it, you can access and manipulate all elements, attributes, and text on the page.

93.) What is `event delegation`?
ans.) Event delegation is a technique where you attach a single event listener to a parent element, instead of attaching separate listeners to each child element. When an event occurs on a child, it bubbles up to the parent, and the single listener can then determine which child triggered the event. This improves performance and simplifies code, especially for dynamically added elements.
eg.) `ul.addEventListener('click', function(event) {`
    `  if (event.target.tagName === 'LI') {`
    `    console.log('List item clicked:', event.target.textContent);`
    `  }`
    `});`

94.) What is `JSON.parse()` and `JSON.stringify()` used for?
ans.)
    - `JSON.parse()`: Converts a JSON string into a JavaScript object.
    - `JSON.stringify()`: Converts a JavaScript object or value into a JSON string.
eg.) `const obj = { name: "Test" };`
    `const jsonStr = JSON.stringify(obj); // '{"name":"Test"}'`
    `const parsedObj = JSON.parse(jsonStr); // { name: "Test" }`

95.) What is `nullish coalescing operator (??)` (ES11, often discussed with ES6+)
ans.) The nullish coalescing operator `??` returns its right-hand side operand when its left-hand side operand is `null` or `undefined`, and otherwise returns its left-hand side operand. It's distinct from `||` which returns the right-hand side for *any* falsy value (0, '', false, etc.).
eg.) `const value = null;`
    `const result = value ?? 'default'; // result is 'default'`
    `const num = 0;`
    `const result2 = num ?? 100;       // result2 is 0 (because 0 is not null or undefined)`
    `const result3 = num || 100;       // result3 is 100 (because 0 is falsy)`

96.) What is `optional chaining (?.)` (ES11, often discussed with ES6+)
ans.) The optional chaining operator `?.` allows you to safely access properties of an object that might be `null` or `undefined` without causing a `TypeError`. If a property in the chain is `null` or `undefined`, the expression short-circuits and returns `undefined`.
eg.) `const user = { profile: { name: "Alice" } };`
    `console.log(user?.profile?.name);   // "Alice"`
    `console.log(user?.address?.street); // undefined (no error)`
    `// Vs. user.address.street would throw an error if address is undefined.`

97.) What is the difference between `let` and `var` in terms of hoisting?
ans.) Both `let` and `var` declarations are hoisted. However, `var` variables are initialized with `undefined` during hoisting, so you can access them before their declaration (though it's bad practice). `let` variables are hoisted but are in a "temporal dead zone" until their declaration is executed, meaning accessing them before declaration results in a `ReferenceError`.
eg.) `console.log(varVariable); // undefined`
    `var varVariable = 10;`
    `// console.log(letVariable); // ReferenceError: Cannot access 'letVariable' before initialization`
    `// let letVariable = 20;`

98.) What is a pure function?
ans.) A pure function is a function that:
    1. Returns the same output for the same given input (deterministic).
    2. Has no side effects (does not modify anything outside its scope, like global variables, console logs, or network requests).
Pure functions are easier to test and reason about.
eg.) `function add(a, b) { return a + b; }` (pure)
eg.) `let total = 0; function addToTotal(num) { total += num; }` (impure, modifies `total`)

99.) What is `Array.prototype.map()`?
ans.) The `map()` method creates a **new array** by calling a provided function on every element in the calling array. It does not modify the original array. It's commonly used for transforming data.
eg.) `const numbers = [1, 2, 3];`
    `const doubled = numbers.map(num => num * 2);`
    `console.log(doubled); // [2, 4, 6]`
    `console.log(numbers); // [1, 2, 3] (original array unchanged)`

100.) What is `Array.prototype.filter()`?
ans.) The `filter()` method creates a **new array** containing all elements from the original array that pass the test (return `true`) implemented by the provided function. It also does not modify the original array.
eg.) `const ages = [12, 18, 20, 25];`
    `const adults = ages.filter(age => age >= 18);`
    `console.log(adults); // [18, 20, 25]`
    `console.log(ages);   // [12, 18, 20, 25] (original array unchanged)`
